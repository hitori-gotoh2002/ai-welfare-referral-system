# 3인 분업 방향

이 프로젝트는 화면, 데이터/API, AI 추천 품질이 서로 맞물려 있습니다. 세 사람이 충돌을 줄이려면 파일과 책임 경계를 명확히 나누는 방식이 좋습니다.

## 역할 1. Product/UX & 현장 워크플로우

목표: 현장 종사자가 상담 중 빠르게 쓰는 화면 흐름을 완성합니다.

주요 책임:

- 상담 입력 화면 UX 개선
- 대시보드/최근 상담/추천서 화면 정리
- 모바일 웹 사용성 점검
- 추천서 문구와 사례기록용 출력 포맷 개선
- 현장 시나리오별 입력 템플릿 작성

주요 파일:

- `index.html`
- `styles.css`
- `app.js` 중 화면 렌더링 함수
- `docs/`의 사용자 흐름 문서

권장 이슈 예시:

- 상담 메모 입력 중 자동저장 표시 추가
- 추천서 인쇄/PDF 레이아웃 개선
- 모바일 메뉴와 패키지 편집 UI 개선
- 문제 유형별 질문 가이드 확장

## 역할 2. Backend/Data & API 품질

목표: 공공데이터/복지로/기관 API를 안정적으로 연결하고 데이터 품질을 높입니다.

주요 책임:

- 공공데이터 API 커넥터 유지보수
- 상세조회 필드 매핑 고도화
- 기관/민간자원 API 지역 필터 개선
- 중복 서비스 정규화
- SQLite 또는 PostgreSQL 저장소 추가
- Render 배포/환경변수 관리

주요 파일:

- `backend_server.py`
- `.env.example`
- `render.yaml`
- 향후 `data/`, `db/`, `services/` 모듈

권장 이슈 예시:

- 중앙부처/지자체 상세조회 필드 매핑 테스트 추가
- 지역명 표준화 테이블 추가
- 서비스 중복 묶음 알고리즘 개선
- SQLite 기반 사례 저장소 도입
- API timeout/cache/retry 정책 추가

## 역할 3. AI/Recommendation & Safety

목표: LLM 구조화와 추천서 생성의 품질을 높이고, 환각과 개인정보 리스크를 줄입니다.

주요 책임:

- Gemini 프롬프트 개선
- LLM JSON 스키마 검증
- 추천서 생성 안전 규칙 강화
- 개인정보/민감정보 마스킹
- 패키지 추천 점수 로직 개선
- LLM 실패/쿼터 초과 폴백 메시지 정리

주요 파일:

- `backend_server.py` 중 `analyze_case`, `build_report`, 패키지 추천 로직
- 향후 `ai/`, `recommendation/`, `safety/` 모듈
- `docs/AI_SAFETY.md`

권장 이슈 예시:

- LLM 응답 JSON schema validation 추가
- 주민번호/전화번호/주소 마스킹 추가
- 추천 근거와 출처 인용 문장 강화
- 긴급 신호 감지 체크리스트 개선
- 패키지 추천 점수표와 테스트 케이스 작성

## 협업 규칙

- `main`: 배포 가능한 안정 버전
- 기능 브랜치: `feature/<area>-<summary>`
- 버그 브랜치: `fix/<area>-<summary>`
- PR은 작은 단위로 만들고, 화면/API/AI 영향 범위를 명시합니다.
- `.env`는 절대 커밋하지 않습니다.
- 실제 개인정보가 들어간 테스트 데이터는 저장소에 올리지 않습니다.

## 권장 마일스톤

### Milestone 1. MVP 안정화

- 입력/메뉴 이동 버그 제거
- API 호출 실패 UX 정리
- 추천서 기본 품질 개선
- Render 배포 성공

### Milestone 2. 저장과 재사용

- SQLite 저장소 도입
- 사례 저장/불러오기 실제화
- 추천서 저장 이력 추가
- 최근 상담 목록 DB 연동

### Milestone 3. 현장 검증

- 3개 시나리오별 테스트 케이스 작성
- 현장 종사자 피드백 반영
- 개인정보 마스킹/권한 설계
- 추천 품질 평가표 작성

### Milestone 4. 운영 준비

- 로그인/권한 추가
- 감사 로그
- 배포 환경 분리
- API 키 재발급 및 제한
